# What worked this week?
- During this week, we improved the accessibility and usability of our system’s Docker images. Previously, 
Docker images were built internally through our CI/CD pipelines but were not publicly accessible. We updated 
the publishing process to push the images to DockerHub, allowing users to easily download and run them as part 
of the system setup.

- This week, we also created new documents such as beta.md, developer-guide.md, and non-developer-guide.md, 
and added additional sections to README.md. While writing these documents, we realized that some installation and 
usage instructions were repeated across multiple files. We reorganized the documentation structure to reduce 
duplication and added cross-references between documents so that users can navigate them more clearly and better 
understand the system setup and usage.

# What did not work this week?

- While testing the application locally, we encountered compatibility issues in our macOS-based setup. Some team 
members experienced difficulties running Ubuntu 24.04.4 LTS in a VirtualBox virtual machine on macOS, while others 
encountered problems when running certain Docker images inside the Ubuntu environment. Since all team members use 
Mac machines, this setup did not reliably support all Docker images required for our system. To ensure consistent 
end-to-end testing, we moved our testing environment to an AWS EC2 instance running Ubuntu 24.04.4 LTS, which provided 
a stable Linux environment for running all services and containers.

- We also experienced failures in our CI/CD workflows due to GitHub Actions artifact publishing limits. The workflows 
exceeded the storage quota for artifacts, causing some runs to fail. After investigating the issue, we removed 
unnecessary artifact uploads and configured the remaining artifact uploads with a 7-day retention policy. This reduced 
storage usage and prevented further CI/CD failures caused by artifact quota exhaustion.
